软件使用说明

这个工具怎么用
这个工具只做本地账号管理：保存你导入过的 Codex 登录备份、显示你手动刷新过的额度、按你点击的按钮切换账号。新增账号时先点“准备登录新账号”。CAS 最终只看 Codex 写出的 auth.json，不看浏览器里登录了几个账号。
切换账号
在账号列表里找到要用的邮箱，点右侧“切换”。确认后会把本机 Codex 登录状态换成这个账号，并自动重启 Codex。切换前先停下正在跑的任务，不要多个账号同时跑同一批任务。
不要在 Codex 里退出账号
这个号以后还要给 CAS 管，就别在 Codex 设置里点退出。退出可能让旧登录凭证失效，CAS 再切回来就会显示异常。新增账号时走“准备登录新账号”。如果默认浏览器打开了错号，可以把 OAuth 登录页地址复制到正确浏览器/Profile 里登录；只要 Codex 最后登录成功，CAS 就能导入。
安全边界
这里只保留手动操作：手动导入、手动切换、手动刷新单个账号。不做反代、不改代理、不自动轮换、不低额度自动换号，也不并发多号跑同一批任务。不要把 CAS 当账号池用，账号多时更要降低登录、切换和刷新频率。
查看额度
每个账号的“模型配额”里有 5H 和 7D 两条血条。它只显示上次手动刷新后的结果，不会在后台自动刷。
刷新额度
想看某个号的最新额度，就只点那一行的“刷新”。不要频繁连续刷，也不要多个号一起刷。
账号备注
点“账号备注”会进入单独的备注页面。每个邮箱一张本地备注卡，可以放登录线索、手机号、密钥提示，不参与登录、切换或刷新。
删除账号
不用的账号可以点“删除”，删除前会再问你一次，防止误删。
